# Функционал (Features)

Детальное описание модулей **Islamic Planner (Barakah Planner)**.

---

## 1. Landing Page (Привлечение)

- 4-х языковый лендинг (RU, KK, AR, EN) с переключением на лету и автоопределением по URL (например `islamic-planer.web.app/kk`).
- Визуально-премиальный дизайн: glassmorphism, градиенты, анимации параллакса.
- Кнопка «Начать бесплатно» перекидывает пользователя прямо в `app.html?lang={current}`.

---

## 2. Дневной Планер (Daily View)

Модуль для ежедневного контроля.

- **Ният (Намерение):** Окошко для напоминания себе, ради чего делается сегодняшняя работа.
- **Namaz Tracker (Трекер Намаза):** Умный виджет в заголовке с обратным отсчётом до следующего намаза. Интеграция с `muftyat.kz`. Поддерживает выбор города, загружает данные на целый год и кэширует в `localStorage`. Включает систему Graceful Fallback (геометрический поиск ближайшего города при падении API по базе 6000+ городов).
- **Ибадат Трекер:** Чекбоксы для намазов (Фаджр, Зухр, Аср, Магриб, Иша). Текущий намаз динамически подчёркивается пульсирующей анимацией.
- **Временные блоки:** Разделение дня на исламские интервалы (От Фаджра до Зухра и т.д.).
- **Умное поле Дата/Хиджра:** Рядом с Григорианской датой отображается Исламский месяц и год.
- **Интерактивные Анимации (Скевоморфизм):**
  - При переключении главных вкладок (День, Календарь, Цели, Доска) срабатывает реалистичная 3D-анимация перелистывания страницы блокнота.
  - При переключении дней (стрелки Вперёд/Назад) происходит анимация «отрывного календаря».
- **AI Planner (Gemini):** Умный ассистент, генерирующий план дня на основе пожеланий пользователя.
- **UX Фишки:** Авторасширяющиеся текстовые поля без полос прокрутки.

---

## 3. Исламский Календарь (Hijra Calendar & Event System)

- Полновесный визуальный календарь-сетка.
- **События (Events/Reminders):** Создание событий 5 типов (Исламские, Личные, Работа, Семья, Здоровье).
- **Напоминания:** Работают через браузерный Push API (бесплатно и оффлайн). С сохранением в Firebase `reminders`.
- **Сунна-посты:** Подсказки на дни Сунны (Понедельник и Четверг) и Джума.

---

## 4. Tafakkur Board (Miro-style Бесконечная Доска) — v2

Наше пространство для свободного размышления, построенное на **Konva.js**.

### 4.1 Множество досок (Board Dashboard)

- Создание неограниченного числа досок.
- Каждая доска: имя (редактируется инлайн), иконка-превью (выбор из 20 вариантов), дата последнего открытия.
- Переход между дашбордом досок и холстом — анимация перелистывания.

### 4.2 Miro-style Вертикальный Тулбар

Плавающий вертикальный тулбар слева с 8 инструментами:

| Кнопка | Инструмент | Shortcut |
| ------ | ---------- | -------- |
| 🖱️ | Pan / Выделение | **V** |
| T | Текст | **T** |
| 🗒️ | Стикер | **N** |
| □ | Фигура (с sub-picker) | **S** |
| → | Стрелка | **A** |
| ✏️ | Карандаш | **P** |
| ⊘ | Ластик | **E** |
| ↩️ / ↪️ | Undo / Redo | **Ctrl+Z / Ctrl+Y** |

Активный инструмент подсвечивается индиго-фоном (`.tool-active`).

### 4.3 Shape Picker (Sub-picker фигур)

Нажатие на кнопку «Фигура» открывает mini-popup с выбором:

- Прямоугольник, Круг, Эллипс, Треугольник, Звезда, Пятиугольник, Шестиугольник, Ромб

После выбора — инструмент активируется и иконка кнопки обновляется.

### 4.4 Context Menu (Плавающий тулбар объекта)

При выборе любого объекта сверху появляется floating toolbar с:

- **7 цветных кружочков** (Жёлтый, Зелёный, Голубой, Розовый, Оранжевый, Белый, Тёмный) — мгновенная смена цвета стикера / фигуры / текста
- **+/−** — увеличение/уменьшение шрифта
- **⫛** — перенести на передний план
- **🗑️** — удалить объект

Активный цвет подсвечивается индиго-рамкой.

### 4.5 Инструменты

| Инструмент | Описание |
| ---------- | -------- |
| **Pan** | Перемещение холста. Stage становится draggable. Cursor: grab. |
| **Текст** | Клик на холст — создаёт текстовый объект. Двойной клик — редактирование через overlay-textarea (позиционируется точно через `getBoundingClientRect`). |
| **Стикер** | Создаёт Konva.Group (Rect + Text). 7 цветов через context menu. Двойной клик — редактирование. |
| **Фигура** | Рисует выбранную геометрическую фигуру протяжкой мыши. |
| **Стрелка** | Рисует стрелку между двумя точками. |
| **Карандаш** | Свободное рисование. `globalCompositeOperation: source-over`. |
| **Ластик** | Стирает линии через `globalCompositeOperation: destination-out`. Размер × 8. |

### 4.6 Навигация

- **Zoom:** Колесо мыши / Pinch. Ограничение: 0.1×–5×.
- **Pan:** Зажать пробел + мышь, или выбрать Pan-tool.
- **Кнопка «Центр»:** Сбрасывает zoom и возвращает в начало холста.

### 4.7 Auto-Save

- Состояние ноды сохраняется в LocalStorage через debounce 1 секунда.
- Рисунки (pathLayer) сохраняются в IndexedDB.

### 4.8 Ограничения и состояние (Known Issues / WIP)

✅ **Исправлено** (март 2026):
- **Z-Index конфликт:** `moveToTop` на Transformer больше не перекрывает `pathLayer`.
- **Позиция textarea при зуме:** Теперь точно позиционируется через canvas `getBoundingClientRect`, без дрейфа при масштабировании.
- **Undo/Redo стабильность:** Исправлены: сохранение нулевых фигур в историю; стейл Transformer-ноды в снапшоте; `selectNode(null)` перед восстановлением.
- **dblclick в Group после undo:** `dblclick` цепляется ко всем текстовым нодам в группе.
- **dragend для всех фигур:** Circle, Ellipse, RegularPolygon, Star теперь тоже сохраняют историю через dragend.

⚠️ **В работе**:
- **Вставка изображений:** Drag-and-Drop и буфер обмена не поддерживаются.
- **Сенсорные экраны:** Требует дополнительного тюнинга на мобильных.
- **Группировка объектов:** Явной мульти-группировки (Shift+click) нет.

## 5. Глобальные Цели (Ахират и Дунья)

Разделение планирования на:

- **Ахират (Вечность):** Изучение Религии, Пожертвования (Садака Джария), Нравственность.
- **Дунья (Мирское):** Дела бизнеса, семья, здоровье, обучение.

Цели отображаются с прогресс-баром и могут быть раскрыты в подпункты.

---

## 6. Activity Log (Аудит действий)

- Журнал действий: «вошёл в систему», «добавил стикер», «сохранил 10 задач» и т.д.
- Мотивирует через Streak-систему. Хранится и локально, и в Firestore (`users/{uid}/activity_log`).

---

## 7. Выживание: Резервное копирование и PWA

- **100% Оффлайн:** PWA ставится как нативное приложение (iOS/Android/Windows).
- **Import / Export Backup:** Выгрузка всей базы (доска, задачи, логи) в один JSON. Если кэш сотрётся — жизнь можно восстановить.
- **Local Mode:** Открытие через `file://` работает без Firebase с PIN-кодом.

---

## 8. Firebase Auth & Cloud Sync

- **Вход:** Google OAuth, Email/Password, либо **Guest Mode** (Продолжить без авторизации) для полностью оффлайн использования.
- **Синхронизация:** При вязке аккаунта данные (задачи, напоминания, активность) синхронизируются с Firestore.
- **Firestore Rules:** Строгая изоляция данных по `uid`. Пользователь видит только свои данные.
- **FCM (Push Notifications):** Токены хранятся в `users/{uid}/fcmTokens`.

## 10. Локализация (i18n)

- 4 языка: **RU** (основной), **KK** (казахский), **EN** (английский), **AR** (арабский, RTL).
- Переключение: кнопка в правом нижнем углу или параметр `?lang=` в URL.
- При переходе между Лендингом (`index.html`) и Приложением (`app.html`), язык надёжно сохраняется и маршрутизируется через защищенный статус `localStorage`, предотвращая сброс на язык по умолчанию.
- Все строки UI переведены через `data-i18n` атрибуты в HTML, которые `i18n.js` заполняет динамически.
