# Техническая Архитектура (SurviveKit)

МашаАллах, наш код чист, прост и надежен как автомат Калашникова. Вот как все устроено под капотом.

## 🏗️ Стек Технологий

- **HTML5:** Семантическая разметка (один файл `index.html` как точка входа).
- **CSS3 / TailwindCSS:** Дизайн через классы Tailwind, кастомные анимации и переменные в `style.css`.
- **Vanilla JavaScript (ES Modules):** Используем нативную систему модулей (`import`/`export`) для чистоты кода. Бандлинг через Vite (`vite-plugin-singlefile`) для сборки в один HTML.
- **Двойное хранилище (Dual Storage):**
  - `LocalStorage`: для быстрых, синхронных данных (тексты задач, опции).
  - `IndexedDB`: для тяжелых бинарных данных (холст Tafakkur Board), чтобы обойти лимит в 5МБ.

## 📂 Структура Файлов

- `index.html` — Оболочка, верстка всех вкладок. Единая точка входа (`<script type="module" src="./main.js">`).
- `style.css` — Кастомные стили (премиальные анимации чекбоксов, сетка для доски, текстура бумаги).
- `store.js` — Слой данных (Data Layer). Экспортирует объект `Store` с методами работы с localStorage и IndexedDB. Содержит логику импорта/экспорта (Backup).
- `board.js` — Движок бесконечного холста (Zoom, Pan, Drag & Drop).
- `main.js` — Контроллер (Controller). Управляет DOM, вкладками и инициализацией.
- `sw.js` & `manifest.json` — PWA Service Worker для 100% оффлайн-доступа и кэширования статики.

## ⚡ Оффлайн-First (SurviveKit & PWA)

Мы перевели проект на современный стек:
**Зачем?**

1. **ES Modules + Vite:** Позволяют писать чистый, модульный код без засорения глобального объекта `window`. При сборке (build) Vite "схлопывает" все в один `index.html` вместе с CSS.
2. **PWA (Service Worker):** Приложение устанавливается на смартфон и кэширует шрифты. Пользователь может открыть его в самолете или в лесу — оно загрузится моментально.
3. **Backup & Restore:** Полный экспорт данных в JSON встроен в `store.js`, гарантируя выживание данных при случайной очистке кэша браузера.

## 💾 Управление Состоянием (State Management)

Все данные сохраняются при любом изменении (ввод текста, клик по чекбоксу).

- Ключ `barakah_day_YYYY-MM-DD` — для дневных планов.
- Ключ `barakah_goals` — для глобальных целей.
- Ключ `barakah_board_state` — для сохранения стикеров на доске (координаты X, Y, текст).
