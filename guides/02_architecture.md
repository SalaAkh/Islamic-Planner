# Техническая Архитектура (SurviveKit)

МашаАллах, наш код чист, прост и надежен как автомат Калашникова. Вот как все устроено под капотом.

## 🏗️ Стек Технологий

- **HTML5:** Семантическая разметка (один файл `index.html` как точка входа). Поля ввода реализованы через `<textarea>` для поддержки многострочного текста.
- **CSS3 / TailwindCSS:** Дизайн через классы Tailwind, кастомные анимации и переменные в `style.css`.
- **Vanilla JavaScript (ES Modules):** Используем нативную систему модулей (`import`/`export`) для чистоты кода. Бандлинг через кастомный Node.js скрипт `build-monolith.js`, который инлайнит все стили и скрипты в один `index.html`.
- **Тройное хранилище (Triple Storage) & Sync:**
  - `LocalStorage`: для быстрых, синхронных данных в оффлайне (тексты задач, опции).
  - `IndexedDB`: для тяжелых бинарных данных (холст Tafakkur Board).
  - `Firebase Firestore`: для облачной синхронизации данных между устройствами (через `db.js`).

## 📂 Структура Файлов

- `index.html` — Оболочка, верстка всех вкладок. Единая точка входа (`<script type="module" src="./main.js">`).
- `style.css` — Кастомные стили (премиальные анимации чекбоксов, сетка для доски, текстура бумаги, auto-resize textarea).
- `store.js` — Слой данных (Data Layer). Экспортирует объект `Store` с методами работы с localStorage и IndexedDB. Содержит логику импорта/экспорта (Backup).
- `board.js` — Движок бесконечного холста (Zoom, Pan, Drag & Drop). Поддерживает стикеры 4-х цветов: Желтый, Зеленый, Синий, Розовый.
- `i18n.js` — Система локализации. Содержит словари на 4 языка и логику бесшовного переключения DOM через `?lang=` параметры (SPA подход без перезагрузки страниц).
- `build-monolith.js` — Кастомный скрипт сборки. Превращает исходники в единый автономный `dist/index.html`.
- `main.js` — Контроллер (Controller). Управляет DOM, вкладками, таймерами доната, инициализацией и auto-resize для textarea полей.
- `sw.js` & `manifest.json` — PWA Service Worker для 100% оффлайн-доступа и кэширования статики.
- `firebase-init.js`, `auth.js`, `db.js` — Интеграция Firebase (Google/Email авторизация, синхронизация Firestore).
- `ai-planner.js` (опционально) — Модуль интеграции с Google Gemini API.

## ⚡ Оффлайн-First (SurviveKit & PWA)

Мы перевели проект на современный стек:
**Зачем?**

1. **Кастомный Monolith Builder (`build-monolith.js`):** Мы отказались от тяжелых сборщиков в пользу своего скрипта. Он берет исходники и собирает их в *один единственный* `dist/index.html` (включая Tailwind и все JS модули). Это дает максимальную скорость загрузки и 100% автономность.
2. **PWA (Service Worker):** Приложение устанавливается на смартфон и кэширует шрифты и сам монолитный HTML. Пользователь может открыть его в самолете или в лесу — оно загрузится моментально.
3. **Backup & Restore:** Полный экспорт данных в JSON встроен в `store.js`, гарантируя выживание данных при случайной очистке кэша браузера.

## 💾 Управление Состоянием (State Management)

Все данные сохраняются при любом изменении (ввод текста, клик по чекбоксу, изменение задачи).

- Ключ `barakah_board_state` — для сохранения стикеров на доске (координаты X, Y, текст, цвет).
- Динамически добавленные задачи в блоки дня сохраняются в LocalStorage с привязкой к своему блоку-контейнеру и восстанавливаются при перезагрузке страницы.

*Примечание:* Как только пользователь входит в аккаунт через Google или Email (через `auth.js`), методы сохранения в `store.js` автоматически дублируют данные в облако Firebase (коллекция `users/{uid}/data/...`). При загрузке страницы происходит скачивание актуальных данных с облака, если локальные ключи пусты или устарели.

## 🏠 Локальный Режим (Local File Access)

Для работы через `file://` протокол (без хостинга) реализован специальный режим:

- **Автодетекция:** `auth.js` определяет протокол `file://` и пропускает Firebase авторизацию.
- **Локальный пароль:** При открытии через `file://` показывается кастомный промпт с паролем (вместо Firebase Auth модала). Захардкоженный хэш пароля хранится в `auth.js`.
- **Индикатор Local Mode:** В шапке отображается бейдж "🏠 Local Mode", сигнализируя пользователю о режиме работы.
- **CORS-обход:** Все необходимые Firebase SDK загружаются через CDN, а локальные данные читаются напрямую из LocalStorage.

## 🔐 Google Auth и Безопасность

Для обеспечения безупречного входа через Google мы применили следующие решения:

1. **signInWithPopup вместо Redirect:** Мы отказались от `signInWithRedirect` в пользу попапов. Это критично для надежной работы на `localhost` и в некоторых мобильных браузерах, где редиректы могут блокироваться или терять сессию.
2. **Заголовки COOP (Cross-Origin-Opener-Policy):** В `firebase.json` настроен заголовок `same-origin-allow-popups`. Без него современные браузеры блокируют `postMessage` от окна Google обратно в приложение, что «убивает» процесс авторизации.
3. **Authorized Domains:** В Google Cloud Console (OAuth Client) и Firebase Auth прописаны все три домена: `localhost`, `firebaseapp.com` и `web.app`.
4. **Защита изображений:** Фото профиля и контентные изображения защищены от скачивания через комбинацию CSS (`pointer-events: none`, `user-select: none`) и JS-блокировщиков (`contextmenu`, `dragstart`).
